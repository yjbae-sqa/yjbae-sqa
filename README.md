특히, 복잡한 QA 상황을 정리하고 테스트 자동화, CI/CD 기반 품질 검증 체계를 구조적으로 구축해왔습니다.  
조직에서 QA 프로세스 정리, 테스트 자동화 체계 구축, CI/CD 기반 품질 검증이 필요하시다면 [LinkedIn](https://www.linkedin.com/in/yjbae/)으로 연락 부탁드립니다. ☕

## 🧩 Quality Engineering

### CI/CD & Automation
- Docker 기반 테스트 환경에서 자동화 테스트 실행 구조 및 검증 프로세스 설계/운영
- CI/CD 파이프라인과 자동화 테스트를 연동하여 배포 전 검증 프로세스 구축

### Release Quality
- 배포 리스크를 줄이기 위해 품질 지표 기반 Quality Gate 기준 정의 및 검즈 프로세스 정립
- Release 전 핵심 기능 안정성을 확보하기 위해 Smoke 테스트 전략 적용

### QA Process & Technical Testing
- 개발 조직 협업 기반으로 테스트 프로세스 및 배포 검증 흐름 설계/개선
- 서비스 구조와 API 호출 흐름을 기반으로 End-to-End 검증 구조 설계
- 초기 조직에서 QA 프로세스 및 테스트 운영 체계 정의 및 구축

## ✅ Testing
### Service Flow Testing
- 사용자 영향도와 비즈니스 리스크 기준으로 End-to-End 검증 범위 및 시나리오 정의
- API 호출 흐름과 서비스 동작을 기반으로 통합 기능 검증 구조 설계

### API Testing
- 서비스 데이터 흐름과 데이터 정합성을 기준으로 API 검증 포인트 및 응답 검증 기준 정의  
- API Spec 기반 기능 및 통합 검증 구조 설계

### Light Performance Testing
- API 응답 시간 및 처리 한계를 기준으로 성능 검증 범위 정의 및 간단한 부하 테스트 설계
- Newman 기반 API 테스트 실행을 활용한 성능 및 스트레스 테스트 검증 수행

### Test Automation
- 반복 검증 효율성과 유지보수성을 기준으로 UI 테스트 자동화 구조 설계 (Selenium / Appium) 
- TestNG 기반 테스트 실행 구조 및 리포트
